# StackSpend v0.1.0-alpha Checklist

Status: local alpha review checklist for M10/S10-alpha-release.

## Release Boundary

- [x] Local-first CLI, SQLite, and dashboard workflows are the supported alpha path.
- [x] Docker and Compose are for local self-host/dev review only.
- [x] GitHub upload, deploy, push, publish, and hosted SaaS release are out of scope for M10.
- [x] No OAuth, multi-user mode, enterprise FinOps features, or additional providers are included.

## Local Quickstart

- [x] `pnpm install` installs the workspace.
- [x] `pnpm --filter @stackspend/cli dev -- doctor` verifies local configuration shape.
- [x] `pnpm --filter @stackspend/cli dev -- init` creates the local SQLite path.
- [x] `pnpm --filter @stackspend/cli dev -- sync --provider mock` runs without credentials.
- [x] Provider fixture modes use fake JSON under `tests/fixtures/providers`.
- [x] `pnpm --filter @stackspend/web dev` serves the local dashboard.
- [x] `pnpm --filter @stackspend/cli dev -- report daily --lang ko` renders the Korean report without Slack.
- [x] Slack delivery requires an explicit `--send slack` flag and `SLACK_WEBHOOK_URL` in the process environment.

## Docker Local Review

- [x] `Dockerfile` builds from the workspace without embedding secrets.
- [x] `.dockerignore` excludes `.env`, `.env.*`, local SQLite data, logs, build output, and dependencies.
- [x] `compose.yaml` sets only non-secret local defaults and fake fixture paths.
- [x] Compose persists SQLite data in the `stackspend_data` volume at `/data/stackspend.sqlite`.

## Security Gate

- [x] `.env.example` contains blank secret values and fake fixture paths only.
- [x] Provider connectors remain read-only in v0.1.
- [x] Raw provider payloads are normalized/redacted before persistence.
- [x] SQLite stores local normalized snapshots only and must not be committed.
- [x] No telemetry is enabled by default.
- [x] No real credentials, tokens, account IDs, project IDs, invoice IDs, emails, card data, billing profiles, or webhook URLs are included in alpha docs or fixtures.

## Required Local Validation

Run before review:

```bash
pnpm test
pnpm typecheck
git diff --check
git diff --name-only --diff-filter=ACM -z | xargs -0 rg -n -e 'sk-[A-Za-z0-9_-]{8,}' -e 'hooks\.slack\.com/services/[A-Za-z0-9/_-]+' -e 'acct_[A-Za-z0-9_-]{6,}' -e 'project_[A-Za-z0-9_-]{6,}' -e 'invoice_[A-Za-z0-9_-]{6,}' -e '[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}'
```

If Docker is available and local policy allows it:

```bash
docker build --pull=false --target verify -t stackspend:m10-verify .
```
